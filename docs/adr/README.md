# Architecture Decision Records (ADRs)

This folder holds short records of important architecture decisions.

Each ADR captures **one decision**: the context, the choice, and the consequences
(what we gain and what we give up). ADRs are **append-only** — we do not rewrite an
old one when a decision changes; we add a new ADR and mark the old one *Superseded*.

- File name: `NNNN-short-title.md` (for example `0001-use-in-memory-store.md`).
- Format: based on Michael Nygard's template (Status, Context, Decision, Consequences).
- Write an ADR when a decision is **hard to reverse** or **likely to be questioned later**.
